1Tier 1
How do encrypted backups reach iCloud?
Option B — app-encrypted blobRECOMMENDED
Store a passphrase-encrypted ncryptsec in your private iCloud. Apple sees only ciphertext; the passphrase never leaves the device. Working key stays device-only.
Option A — iCloud Keychain sync
~5 lines, no passphrase, auto multi-device. But the raw key enters Apple's escrow — contradicts "never leaves the device."
No iCloud — manual only
Seed words / ncryptsec / QR only. Zero cloud exposure, but no auto recover-a-lost-phone path.
Why it matters: defines the core trust story and how much gets built.
2Tier 1
Switch new-key generation to a seed phrase (NIP-06)?
Yes — generate from a BIP-39 mnemonicRECOMMENDED
Enables 12/24-word backup & restore — the format most Nostr users expect. Changes Keys.generate() for future keys only.
No — keep raw key generation
Simpler, but seed words are impossible — backup is ncryptsec/iCloud only, forever.
If yes — keep the mnemonic around?
Persist it encryptedRECOMMENDED
Lets "show my seed words later" + seed-based iCloud work, under one passphrase.
Show once, then discard
Smallest secret footprint, but words can never be re-displayed.
Why it matters: seed words can't be added to a key after the fact — this is a now-or-never call for new keys.
